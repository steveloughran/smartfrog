SmartFrog™ Eclipse Plug-In Release Notes

Feature list

  1. Project management with SmartFrog project wizard
  2. Component and description wizards for creating SmartFrog component and description files
  3. Specialized SmartFrog description file editor
  4. Pop-up source code help
  5. Test and debug capabilities
  6. Loose integration with the SmartFrog GUI
  7. Integrated online help
  8. Context sensitive help

Limitations

  1. The SmartFrog Eclipse plug-in automatically deploys description files only. If you want to deploy component files, you need to manually generate the skeleton and stub files in the output directory and add the output directory to the sfExamples.jar file.
  2. The SmartFrog Eclipse plug-in can deploy description files to the local machine only. The process name is fixed and is called “process.” If you want to change the process name and deploy location, you must use the SmartFrog GUI, which is loosely integrated into the SmartFrog Eclipse plug-in.
  3. On the SmartFrog Eclipse plug-in toolbar, the “Stop Application” button is not fully implemented. You can use the “Stop SmartFrog daemon” button on the toolbar to kill the daemon.

These limitations will be addressed in our next release.

Know Problem

On Linux systems, the default browser for online help is Mozilla. You can do one of the following to solve this problem:

  1. You can install Mozilla on your system. It is available at http://www.mozilla.org.
  2. You can change the Help preferences in Eclipse.

    To change the Help preferences in Eclipse:
    1. Select Window > Preferences > Help from the Eclipse menu bar. The Preferences dialog shows Custom Browser (user defined program) with mozilla %1 in the Custom Browser command field.
    2. Change the Custom Browser command field as follows:
    • If you are using Firefox (also available at http://www.mozilla.org), change the value in the Custom Browser command field to firefox %1.
    • If you are using KDE, change the value in the Custom Browser command field to kfmclient openfile webbrowsing %1.
    • If you are using Gnome, change the value in the Custom Browser command field to evolution %1.
