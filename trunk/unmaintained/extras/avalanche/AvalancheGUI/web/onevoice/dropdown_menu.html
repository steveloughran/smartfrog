HTML/CSS

  • At the most basic level, a dropdown menu consists of div tags of class "globalNavTrigger". These divs make up the top level triggers and each of them contains an unordered list element whose items represent their menu items.
  • Looking within these unordered lists, the li elements can have their own submenus by being given class "subMenuTrigger" and containing another unordered list element of class "dropdownMenu".
  • There is a unique id tagging system that keeps menus and triggers associated with eachother. the class="dropdownMenu" item is given an id unique to the page, and it's trigger li element is given a matching id suffixed with "_trigger".
  • Within menu items that do not trigger further submenus, the menu item label is wrapped in an anchor tag. In this way links can drive navigation in either the main window through straightforward href and target attributes, or JavaScript can be executed by menu items through the use of the "javascript:" protocol in the href attribute.

Javascript

  • DropdownMenuManager.js contains a number of methods to both initialize the components when loaded and to manage their visibility, positioning and highlighting changes at runtime.
  • Most notably, a large init function whose execution is managed by the reconcileEventHandlers function within global.js, attaches all of the various event handlers required to power the component.
  • Developers can refer to comments within the DropdownMenuManager code itself for further comments.
