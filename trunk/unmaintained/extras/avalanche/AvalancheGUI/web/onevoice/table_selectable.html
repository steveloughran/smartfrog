HTML/CSS

  • This is essentially the same component as the basic table, with the following differences:
    • The table is given a unique id attribute, (unique to any given single view , not necessarily across the entire application)
    • each row is given a checkbox or radio-button cell, and the checkbox/radio-button is given the attribute "rowselector", whose value is "yes".
    • the column caption for the checkbox cell contains another checkbox cell, and the checkbox is given the attribute "tableid", whose value is a unique id matching the unique id of the table.
    • For the Non-Explicit Selectable Table, there is no seperate cell for checkboxes or radio buttons. Instead, an input element of type="hidden" is placed within the first cell of the table. This hidden field will hold a value of either "1" or "0" (which states correspond to the checked and unchecked states of checkboxes in the explicit selectable tables).

Javascript

  • The tableManager script will automatically look for these input elements, make sure they have a rowselector attribute whose value is "yes". When it finds one it will attach the necesary event handlers to handle row selection and deselection.
