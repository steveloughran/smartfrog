Overview of the SmartFrog Eclipse plug-in

SmartFrog™ (Smart Framework for Object Groups) is a framework for configuring and automatically activating distributed applications. SmartFrog is both a language for describing component collections and component configuration parameters and a runtime environment, which activates and manages these components. In this model, a component is a Java application that has implemented the SmartFrog interface APIs, which support various lifecycle operations. The Java component is described using the SmartFrog description language notation. These descriptions are contained in SmartFrog description files, which specify component definitions, associations and relationships between the components, and workflows associated with the lifecycle of the components and the system as a whole.

The SmartFrog Eclipse plug-in provides the user with one common integrated development environment (IDE) in which to develop SmartFrog description files and Java component files. The Eclipse platform offers the benefits of project management, version control, and sophisticated Java development tools for writing, compiling, testing, and debugging Java applications.

In the Tasks section of the online help, we use the counter example located in the SmartFrog_Install_Dir\src\org\smartfrog\examples\counter directory to explain how to use the plug-in to create and deploy SmartFrog description and component files. The counter component is relatively simple. It generates a counter with a given seed value and increments it by one untill it reaches the limit attribute value specified in the description file. We selected a simple example by design so that you are not caught up in application details but rather focused on the description and component files and their interaction.

About Eclipse

Eclipse is an open extensible framework that uses a plug-in architecture for integrating development tools. The Eclipse platform is structured around the concept of extension points. Extension points are well-defined places in the system where tools (called plug-ins) can extend and contribute new functionality to the platform.

The Eclipse framework includes:

  • Eclipse platform - basic workbench
  • Java Development Toolkit (JDT) — full featured Java development environment
  • Plug-in Developer Environment (PDE) — specialized tools to aid in the development of plug-ins

Recommended reading

The Eclipse platform comes with the following online books that you can access as part of the available online help:

  • Workbench User Guide - provides basic end user information about using the platform
  • Java Development User Guide - provides information on how to use the platform to develop Java applications
  • Platform Plug-in Developer Guide - provides information for the plug-in developer to extend the platform
  • JDT Plug-in Developer Guide - provides information about the extension points and API provided by the JDT that allow the Java developer to write, compile, test, debug, and edit Java applications
  • PDE Guide - provides information about a set of platform extension contributions (views, editors, perspectives, etc.) that collectively allow the plug-in developer to streamline the process of developing plug-ins inside the workbench

Assumptions

To use this plug-in, you must be familiar with the following:
  • Java development
  • Eclipse IDE
  • Principles of SmartFrog technology
