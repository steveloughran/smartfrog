HTML/CSS

Tree Tables without disclosure controls have the following features:
  • the table tag has class "treeTable".
  • Rows that are at the top-level of the tree's hierarchy are of class "treeTableTopLevel".
  • Nesting is indicating purely by a class of "nested1", "nested2", "nested3" on the first td element in the row.

Tree Tables with disclosure controls use additional HTML and CSS as follows:

  • Rows that are parents of individual trees are of class "treeTableTitleRowOpen", or "treeTableTitleRowClosed".
  • Rows that are parents and that are also at the top-level of the tree's hierarchy are given the additional class name "treeTableTopLevel". The two classnames are seperated by a space.
  • These parent rows must also have a unique id attribute prefixed by "tc_". eg id="tc_7890". Unique id's can be any length
  • Within these rows, the "tree cell" contains two divs:
    • a div of class "treeControl" that is empty
    • a div of class "treeTitle", containing the text for the title.
  • Children of a given individual tree are of class "contentsRow".
  • All child rows in tree tables must have a "parentid" attribute.
  • For children that are only nested one level deep, the value of their parentid attribute matches the id of their parentRow. eg parentid="tc_7890".
  • On the other hand for children nested more than one level deep, the multiple unique id's of its various parent rows are appended and seperated by underscores. For instance a row with parentid="tc_4567_7890" is the child of two trees; one with id "tc_4567", and one with id "tc_7890". Note that the sequence of concatenated id's need not match the hierarchy.
  • horizontal indentation to visually denote the nesting relationship between rows is handled by a seperate layer of class naming. Specifically, within a given row, the relevant Td element (the td whose column contains your treeTitles and treeControls) is given either class "nested1", "nested2", "nested3", etc..
  • Note that the main row of a tree containing the treeTitle and treeControl always shares the nesting level (and thus nesting classname) of its siblings, not of its children.
  • To load a given branch of a Tree Table in its closed state, two things must change.
    • The main row must be of class "treeTableTitleRowClosed" instead of "treeTableTitleRowOpen".
    • Make sure to give all of its child rows a style attribute of style="display:none;"

Javascript

  • treeTableManager.js contains a number of methods to both initialize these components when loaded and to manage their state at runtime.
  • Most notably, a large init function whose execution is managed by the reconcileEventHandlers function within global.js, attaches all of the various event handlers required to power the component.
  • For further discussion about the Selectable Tree Table variation, developers can refer to documentation under Selectable Table.
