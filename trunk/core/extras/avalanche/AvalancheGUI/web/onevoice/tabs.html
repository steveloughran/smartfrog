HTML/CSS

  • Each tab consists of a single link wrapped within a div whose class is either "tabOff" or "tabOn".
  • Sets of tabs are contained within a div whose class is either "tabSet" or "secondaryTabSet", depending on which variation you are implementing.
  • there is a following div currently called "tabBottomLine", that is necessary to clear the CSS float:left layout used by the tabs.
  • the only implementation difference between primary and secondary tabs is that secondary tabs are contained in a particular div. Currently that div has the class "secondaryTabSet". Visual differences are then taken care of by the css and js.

Javascript

  • TabManager.js contains a method to initialize the tabs when they are loaded
  • Tab initialization consists of two parts
    1. The background of the tabs is made clickable, essentially by rerouting the event to the anchor element within.
    2. On Internet Explorer 6, if the width of a tab is less than that specified in the design center, it is set the appropriate width at this time. If the width is already greater than the width specified in the design center no action is taken. Note that this resizing script is not executed by Mozilla clients as Mozilla understands the min-width CSS properties that are set for tabs within the css.
