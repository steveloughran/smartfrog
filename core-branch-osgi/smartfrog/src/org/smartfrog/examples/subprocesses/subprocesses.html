Subprocesses

Goal of the example:

This example explains the concepts of sub-processes through counter implementation.

Internal Functionality:

The subprocesses example has a trace component that traces the various phases of all the components, display component to show the tree of components with trace results, a management component that dispalys the deployment information of all the components and a system component to be deployed. System component has sub-components that are deployed in separate processes. Each sub-component:
    - Generates a counter from the seed value to the specified limit.
    - Has a trace component which traces and displays a message indicating the phases it goes through into the main trace info window.
    - Has a display component that shows the output of the counter component in a separate display window.
The description file subprocess.sf describes these components.

Steps to run the example:
    - Start daemon on localhost
    - Start subprocess.sf
    - Stop daemon

Expected Output:

The root process creates a trace info window, display info window and spawns various subprocesses for the sub-components of system component. The output of all the sub-components are displayed in their respective display window.
