HTML/CSS

  • Transfer Box Components are made up of the following elements:
    1. A form tag at the outside of the component, of class "transferBox", which has been given a unique id attribute. (It will also require method and action attributes as necessary to fit into your application.)
    2. A select element of class "itemCategory". This is to allow the users to filter the available items list by category.
    3. a hidden iframe element that is used to refresh the control's data.

Javascript

  • TransferBoxManager.js contains a number of methods to both initialize the components when loaded and to manage their state at runtime.
  • Most notably, a large init function whose execution is managed by the reconcileEventHandlers function within global.js, attaches all of the various event handlers required to power the component.
  • Developers can refer to comments within the TransferBoxManager code itself for further comments.
  • the document that loads within the hidden iframe should call the method "transferBox_resetAvailableItems" from its parent window. Two arguments are passed to this function, the first being the unique id of the form element, and the second being a concatenated list of all available items.
  • Within this concatenated list, item names and item categories are seperated by "*", which pairs are then seperated from their ids by "|", and then the triplets of of items, categories and id's are then seperated from eachother by ":".
