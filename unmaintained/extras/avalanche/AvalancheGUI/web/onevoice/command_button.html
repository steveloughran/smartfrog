HTML/CSS
  • HTML for a Command Button consists at a basic level of an input or button element, and a set of divs wrapping it.
  • The element is either a button or input element (inputs being of either type="text" or type="submit"), and the element's class is one of "hpButton","hpButtonSmall" or "hpButtonVerySmall".
  • As for the 3 divs, the outermost of these is of class "bWrapper", and the two inner divs are classless.
  • These three nested divs are necessary to create the 3 pixels of slanted border treatment that is visible on the right and bottom edges. Refer to the CSS itself for further information.
  • For the Image Buttons, only the button tag can be used, with class "hpButtonSmall", and an additional classname, "shrinkWrapButton". This class forces a width of 22px, and allows an 11px high image to be contained within.
Javascript
  • ButtonManager.js contains a number of methods to both initialize buttons when the page is loaded and to manage their state changes at runtime.
  • Most notably, a large init function whose execution is managed by the reconcileEventHandlers function within global.js, attaches all of the various event handlers required to power the component.
  • State changes at runtime are visually effected primarily by changes to the className property of the outermost wrapper div.
  • Developers can refer to comments within the ButtonManager code itself for further comments.
  • Although the onclick event handler is available to perform all the usual functions that buttons need to perform, note that the onmousedown, onmouseup, onmouseover and onmouseout event handlers are off limits to applications. As currently implemented these event handlers must be reserved for use by the internal functionality of the button itself.
  • In rare cases where buttons must be arranged in a vertical row, you must wrap the block of buttons in a div of class "verticalButtonSet". This will slightly modify the float and margin properties to revert the buttons to normal block element behaviour
